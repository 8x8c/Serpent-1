Serpent Encryptor

This is a pure Rust implementation of a dummy Serpent‑like encryption tool. It demonstrates the structure of a block cipher using a simplified version of the Serpent algorithm with CBC mode and PKCS7 padding.

Features

  • Dummy Serpent encryption and decryption
  • CBC mode with a hard-coded key and IV
  • PKCS7 padding for data alignment
  • Self-contained implementation with no external crypto dependencies

Usage

The program operates in two modes: encryption (enc) and decryption (dec).

Building

cargo build --release

Encryption

To encrypt a file, run:

cargo run -- enc <input file> <output file>

Example:

cargo run -- enc plain.txt encrypted.bin

Decryption

To decrypt a file, run:

cargo run -- dec <input file> <output file>

Example:

cargo run -- dec encrypted.bin decrypted.txt

Project Overview

The code is implemented entirely in Rust using only the standard library. Key components include:

  • PKCS7 Padding: Adds and removes padding to ensure the data size is a multiple of 16 bytes.
  • Dummy Key Schedule: A simplistic derivation of round keys from a hard-coded 16‑byte key.
  • Dummy Round Functions: Simulated S‑box (bit reversal, which is self‑inverse) and a linear transformation (rotate left by 3 bits with an inverse) that are applied over 32 rounds.
  • CBC Mode: Implements chaining where each block is XORed with the previous ciphertext block (or IV).

Hard-Coded Key and IV

The following values are hard-coded in the source code as 32‑character hexadecimal strings (16 bytes each):

  • Key: 00112233445566778899aabbccddeeff
  • IV: ffeeddccbbaa99887766554433221100

You can modify these values in the source code if needed.

Disclaimer

This implementation is for demonstration purposes only and does not represent a secure, production-ready cipher. The dummy functions and key schedule are highly simplified and should not be used for any real cryptographic applications.
